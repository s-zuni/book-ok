1. Authentication & User Management
Supabase Auth Integration: * Verify that login session persistence and logout flows are handled correctly via context/AuthContext.

Trigger: Ensure a DB Trigger is configured to synchronize data between auth.users and the public.users (or profile) table upon sign-up.

Row Level Security (RLS) Policies: * Confirm RLS is enabled for all tables.

Validate that policies based on auth.uid() are correctly applied so users can only view or modify their own data.

2. My Page & Child Profiles
Children Table Schema: * Verify the existence of parent_id (UUID, Foreign Key), name, birthdate, and type columns.

Ensure the type column values align with the Enums defined in the code (e.g., Infant, Toddler, Lower Elementary, Upper Elementary).

Data Access Permissions: * Test permission policies to ensure parent users can perform CRUD operations (SELECT, INSERT, UPDATE, DELETE) only on their own children's profiles.

3. Book Data & Recommendations
API Route (/api/recommendations): * Confirm that the TTBKey, required for communicating with the Aladin API, is correctly configured in environment variables (.env.local).

Ensure response data correctly maps to the frontend Book interface (e.g., id, title, imgsrc).

Books Table (Caching Logic): * Since read_books references the books table, ensure that when a book is marked as "read" or "saved," the book info is stored in the Supabase books table.

Verify that the logic uses an UPSERT operation to prevent duplicate entries in the books table.

4. Reading History
Read_books Table: * Validate columns: child_id, book_id, and read_date.

Foreign Key Constraints: * Ensure child_id correctly references the children table and book_id references the books table to maintain data integrity.

Statistics & Performance: * Check if the query for counting read books (e.g., count: 'exact') on the My Page is optimized (consider indexing for performance).

5. Community
Posts & Comments Tables: * Confirm both tables are properly initialized.

Supabase Storage: * Verify the creation of a storage bucket (e.g., community-images) and check that its access policy (e.g., Public Read) is set correctly for displaying images.

Engagement Logic: * Check whether view counts and likes are handled via client-side updates or Database Functions (RPC). Note: RPC is highly recommended to prevent data race conditions.

6. Environment Variables & Security
Key Protection: * Double-check that sensitive keys like SUPABASE_SERVICE_ROLE_KEY or the Aladin API Key are never exposed to the client side (keep them out of NEXT_PUBLIC_ prefixes).

Only NEXT_PUBLIC_SUPABASE_URL and NEXT_PUBLIC_SUPABASE_ANON_KEY should be accessible to the browser.
